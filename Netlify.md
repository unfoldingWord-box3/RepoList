# Netlify Site Classification Rules

Rules for deciding what to do with each Netlify site in the unfoldingWord account. Produced by analyzing `sheets/categorized_repos_netlify2.csv` against the GitHub repo classifications in `sheets/categorized_repos.csv`.

## Recommendation labels

| Netlify Recommendation  | Netlify Recommendation Reason                                          |
|-------------------------|------------------------------------------------------------------------|
| **Keep Auto Builds**    | Site is healthy and actively used; auto-deploy stays on.               |
| **Disable Auto Builds** | Site can remain live for reference, but stop rebuilding on every push. |
| **Remove Project**      | Delete the Netlify site entirely.                                      |
| **Manual Review**       | Needs human decision before acting — reason is in the Notes column.    |

## Rules

Rules are applied in priority order; the first matching rule wins.

---

### Rule NL1 — Keep Auto Builds: production site with a custom domain

**Condition:** `custom_domain` is non-empty.

**Netlify Recommendation Reason:** A configured custom domain means the site is serving real traffic under a branded URL. It must stay live and up to date regardless of how old the last publish was.

**Netlify Recommendation:** Keep Auto Builds.

---

### Rule NL2 — Remove Project: orphaned site with no linked repo and no custom domain

**Condition:** `repo_url` is empty AND `custom_domain` is empty AND the site name is autogenerated (random hash slug) or is an obvious placeholder (e.g. "trash-delete").

**Netlify Recommendation Reason:** No repo means the site cannot be rebuilt or updated. An autogenerated or throwaway name confirms it was never a real project. There is nothing to preserve.

**Netlify Recommendation:** Remove Project.

---

### Rule NL3 — Manual Review: no linked repo (not caught by NL2)

**Condition:** `repo_url` is empty AND the site was not matched by NL2.

**Netlify Recommendation Reason:** The site cannot rebuild, but it may be serving something intentionally (a static upload, a manual deploy) or may need investigation before removal.

**Netlify Recommendation:** Manual Review — confirm whether the site has a purpose before removing.

---

### Rule NL4 — Manual Review: backing GitHub repo is archived

**Condition:** The GitHub repo linked via `repo_url` has `archived = TRUE` in the repository data.

**Netlify Recommendation Reason:** An archived repo receives no new commits, so auto-deploy is pointless. But the site itself may still be in use or may need to be removed — that decision requires human review.

**Netlify Recommendation:** Manual Review — decide whether to keep the site as a static artifact (disable auto builds), redirect it, or remove it.

---

### Rule NL5 — Manual Review: site is in a non-organization Netlify account

**Condition:** `account_slug` is not `unfoldingword-hvaaits` (i.e. the site lives in a personal or team account rather than the main unfoldingWord account).

**Netlify Recommendation Reason:** Sites outside the organization account are not under shared ownership and may not be accessible to the full team. They should be transferred or removed.

**Netlify Recommendation:** Manual Review — transfer ownership to the organization account, or remove.

---

### Rule NL6 — Manual Review: site has never been published

**Condition:** `published_at` is empty AND `repo_url` is non-empty.

**Netlify Recommendation Reason:** A configured site that has never successfully published is either broken or abandoned. Auto-deploy is consuming build minutes for zero output.

**Netlify Recommendation:** Manual Review — investigate the build failure or delete the project.

---

### Rule NL7 — Keep Auto Builds: published within the last 12 months

**Condition:** `published_at` is within the last 12 months AND no earlier rule fired.

**Netlify Recommendation Reason:** A recent successful deploy indicates the site is actively maintained. Auto-deploy should remain on.

**Netlify Recommendation:** Keep Auto Builds.

---

### Rule NL8 — Disable Auto Builds: name indicates a POC, demo, test, lab, playground, or template

**Condition:** Site name contains `poc`, `demo`, `test`, `lab`, `playground`, `experiment`, or `template` AND no earlier rule fired.

**Netlify Recommendation Reason:** These are experimental or one-off builds. Keeping auto-deploy active wastes build minutes and creates noise. The site may still be useful as a static reference.

**Netlify Recommendation:** Disable Auto Builds. Consider removing if the site has been inactive for more than 24 months.

---

### Rule NL9 — Disable Auto Builds: published more than 18 months ago

**Condition:** `published_at` is more than 18 months ago AND no earlier rule fired.

**Netlify Recommendation Reason:** A site with no recent deploy is no longer being actively developed. Keeping auto-deploy on wastes build minutes on every push to the backing repo.

**Netlify Recommendation:** Disable Auto Builds.

---

### Rule NL10 — Manual Review: published 12–18 months ago (borderline)

**Condition:** `published_at` is between 12 and 18 months ago AND no earlier rule fired.

**Netlify Recommendation Reason:** The site is neither clearly active nor clearly stale. A human should decide whether it is still in use.

**Netlify Recommendation:** Manual Review — confirm whether active development is ongoing.

---

### Rule NL11 — Manual Review: default

**Condition:** No earlier rule matched.

**Netlify Recommendation:** Manual Review — review manually.

---

## Summary

**Keep Auto Builds** (26): 10 production sites with custom domains + 16 sites published within the last 12 months.

**Disable Auto Builds** (41): Sites not published in 18+ months or with POC/demo/test/lab/playground/template names. These sites can remain live for reference without consuming build minutes.

**Manual Review** (10): Sites needing human review — wrong Netlify account (2), archived backing repo (1), no repo linked but not clearly trash (2), never published (3), borderline publish age (2).

**Remove Project** (2): `trash-delete` (throwaway name, no repo) and `willowy-halva-bb9346` (autogenerated name, no repo).
