# Netlify Site Classification Rules

Rules for deciding what to do with each Netlify site in the unfoldingWord account. Produced by analyzing `sheets/categorized_repos_netlify2.csv` against the GitHub repo classifications in `sheets/categorized_repos.csv`.

## Action labels

| Column | Meaning |
|---|---|
| **Keep Auto Builds** | Site is healthy and actively used; auto-deploy stays on. |
| **Disable Auto Builds** | Site can remain live for reference, but stop rebuilding on every push. |
| **Remove Project** | Delete the Netlify site entirely. |
| **Ask** | Needs human decision before acting — reason is in the Notes column. |

## Rules

Rules are applied in priority order; the first matching rule wins.

---

### Rule NL1 — Keep Auto Builds: production site with a custom domain

**Condition:** `custom_domain` is non-empty.

**Reasoning:** A configured custom domain means the site is serving real traffic under a branded URL. It must stay live and up to date regardless of how old the last publish was.

**Recommended action:** Keep Auto Builds.

---

### Rule NL2 — Remove Project: orphaned site with no linked repo and no custom domain

**Condition:** `repo_url` is empty AND `custom_domain` is empty AND the site name is autogenerated (random hash slug) or is an obvious placeholder (e.g. "trash-delete").

**Reasoning:** No repo means the site cannot be rebuilt or updated. An autogenerated or throwaway name confirms it was never a real project. There is nothing to preserve.

**Recommended action:** Remove Project.

---

### Rule NL3 — Ask: no linked repo (not caught by NL2)

**Condition:** `repo_url` is empty AND the site was not matched by NL2.

**Reasoning:** The site cannot rebuild, but it may be serving something intentionally (a static upload, a manual deploy) or may need investigation before removal.

**Recommended action:** Ask — confirm whether the site has a purpose before removing.

---

### Rule NL4 — Ask: backing GitHub repo is archived

**Condition:** The GitHub repo linked via `repo_url` has `archived = TRUE` in the repository data.

**Reasoning:** An archived repo receives no new commits, so auto-deploy is pointless. But the site itself may still be in use or may need to be removed — that decision requires human review.

**Recommended action:** Ask — decide whether to keep the site as a static artifact (disable auto builds), redirect it, or remove it.

---

### Rule NL5 — Ask: site is in a non-organization Netlify account

**Condition:** `account_slug` is not `unfoldingword-hvaaits` (i.e. the site lives in a personal or team account rather than the main unfoldingWord account).

**Reasoning:** Sites outside the organization account are not under shared ownership and may not be accessible to the full team. They should be transferred or removed.

**Recommended action:** Ask — transfer ownership to the organization account, or remove.

---

### Rule NL6 — Ask: site has never been published

**Condition:** `published_at` is empty AND `repo_url` is non-empty.

**Reasoning:** A configured site that has never successfully published is either broken or abandoned. Auto-deploy is consuming build minutes for zero output.

**Recommended action:** Ask — investigate the build failure or delete the project.

---

### Rule NL7 — Keep Auto Builds: published within the last 12 months

**Condition:** `published_at` is within the last 12 months AND no earlier rule fired.

**Reasoning:** A recent successful deploy indicates the site is actively maintained. Auto-deploy should remain on.

**Recommended action:** Keep Auto Builds.

---

### Rule NL8 — Disable Auto Builds: name indicates a POC, demo, test, lab, playground, or template

**Condition:** Site name contains `poc`, `demo`, `test`, `lab`, `playground`, `experiment`, or `template` AND no earlier rule fired.

**Reasoning:** These are experimental or one-off builds. Keeping auto-deploy active wastes build minutes and creates noise. The site may still be useful as a static reference.

**Recommended action:** Disable Auto Builds. Consider removing if the site has been inactive for more than 24 months.

---

### Rule NL9 — Disable Auto Builds: published more than 18 months ago

**Condition:** `published_at` is more than 18 months ago AND no earlier rule fired.

**Reasoning:** A site with no recent deploy is no longer being actively developed. Keeping auto-deploy on wastes build minutes on every push to the backing repo.

**Recommended action:** Disable Auto Builds.

---

### Rule NL10 — Ask: published 12–18 months ago (borderline)

**Condition:** `published_at` is between 12 and 18 months ago AND no earlier rule fired.

**Reasoning:** The site is neither clearly active nor clearly stale. A human should decide whether it is still in use.

**Recommended action:** Ask — confirm whether active development is ongoing.

---

### Rule NL11 — Ask: default

**Condition:** No earlier rule matched.

**Recommended action:** Ask — review manually.

---
